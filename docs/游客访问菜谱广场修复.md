# 游客访问菜谱广场修复

## 问题描述
游客访问菜谱大全页面时，页面没有展示任何菜谱。

## 问题原因
1. `Recipes.tsx` 中 `recipeTab` 的默认值设置为 `'my'`（我的菜谱）
2. 游客没有"我的菜谱"，`myRecipes` 为空数组
3. 页面默认显示空的 `myRecipes`，导致游客看不到任何菜谱

## 解决方案
1. **修改默认标签页**：游客默认显示 `'public'`（菜谱广场），登录用户默认显示 `'my'`（我的菜谱）
2. **添加用户状态监听**：当用户状态改变时（如退出登录变成游客），自动切换到菜谱广场
3. **隐藏管理按钮**：游客无法看到"分类"、"批量导入"、"创建菜谱"等管理按钮

## 修改文件
- `pages/Recipes.tsx`

## 修改内容
```typescript
// 1. 根据用户状态设置默认标签页
const [recipeTab, setRecipeTab] = useState<'my' | 'public'>(!user || user.isGuest ? 'public' : 'my');

// 2. 监听用户状态变化
useEffect(() => {
  if (!user || user.isGuest) {
    setRecipeTab('public');
  }
}, [user]);

// 3. 只对登录用户显示管理按钮
{user && !user.isGuest && (
  <div className="flex gap-2 flex-wrap md:flex-nowrap">
    {/* 分类、批量导入、创建菜谱按钮 */}
  </div>
)}
```

## 测试验证
- ✅ 游客访问菜谱大全页面，能看到所有公共菜谱
- ✅ 游客看不到"我的菜谱"标签页
- ✅ 游客看不到管理按钮（分类、批量导入、创建菜谱）
- ✅ 登录用户默认显示"我的菜谱"
- ✅ 登录用户可以切换到"菜谱广场"查看所有菜谱
